# Phase 7 Plan (Detailed)

Date: 2026-02-24

## Objective

Execute a concrete bidirectional interoperability matrix between `casacore` and
`casacore-mini` for currently implemented persistence surfaces, using
structure-level read/verify checks as the pass/fail gate. Canonical ASCII dumps
remain available for diagnostics when a verify step fails.

## Scope for this wave

In scope (scoped prove-out):

1. binary `AipsIO` `Record` artifacts generated by both systems
2. `table.dat` top-level header artifacts generated by both systems
3. 2x2 producer/consumer matrix validation using both systems' semantic
   verifiers

Out of scope (deferred to later Phase 7 waves):

1. full `table.dat` body (`TableDesc`, column descriptors, manager metadata)
2. storage-manager payload file interop (`table.f*`)
3. full MeasurementSet/image tree bidirectional artifact generation

## Interop matrix

For each scoped artifact:

1. Producer: `casacore`; Consumer: `casacore`
2. Producer: `casacore`; Consumer: `casacore-mini`
3. Producer: `casacore-mini`; Consumer: `casacore`
4. Producer: `casacore-mini`; Consumer: `casacore-mini`

All matrix cells must pass typed semantic verification for the artifact kind.

## Tooling layout

1. `tools/interop/casacore_interop_tool.cpp`:
   - writes scoped artifacts with `casacore`
   - verifies scoped artifacts against expected logical structures with
     `casacore`
   - emits canonical dumps for diagnostics
2. `src/interop_mini_tool.cpp`:
   - writes scoped artifacts with `casacore-mini`
   - verifies scoped artifacts against expected logical structures with
     `casacore-mini`
   - emits canonical dumps for diagnostics
3. `tools/interop/run_phase7_matrix.sh`:
   - builds tools
   - executes matrix
   - on verify failure, emits both consumer dumps into `build*/phase7_artifacts/`

## Local prerequisites

`casacore` must be installed and discoverable via `pkg-config` and PATH:

```bash
brew tap casacore/tap
brew install casacore
```

Expected commands available after install:

1. `taql`
2. `showtableinfo`
3. `pkg-config --modversion casacore`

## Acceptance criteria for this wave

1. `tools/interop/run_phase7_matrix.sh` passes on local macOS dev environment.
2. Both producer outputs are consumable by both readers for all scoped
   artifacts.
3. Semantic verification (not raw binary equality) is the primary gate.
4. Diagnostic dumps are generated automatically on verify failure.
5. Any known unsupported types/surfaces are explicitly documented in exit notes.
